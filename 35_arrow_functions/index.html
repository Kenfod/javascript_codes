Arrow Functions

       Arrow functions, introduced in ES6 (ECMAScript 2015), 
       provide a concise syntax for writing function expressions in JavaScript. 
       They offer a shorter alternative to traditional function expressions 
       and have distinct characteristics, particularly regarding the "this" keyword.

        Syntax:
        The basic syntax of an arrow function is:

        (parameters) => expression

        or for a multi-line function body:

          (parameters) => {
            // function body
            return value;
          }

        Key Features:
        Concise Syntax: Arrow functions allow for a more compact way to write functions, 
        especially for single-expression functions where the "return" keyword and 
        curly braces can be omitted for an implicit return.

              // Traditional function expression
              const sum = function(a, b) {
                return a + b;
              };

              // Arrow function with implicit return
              const sumArrow = (a, b) => a + b;

        Lexical "this" Binding: Unlike regular functions, arrow functions do not have their own "this" binding.
        Instead, they inherit "this" from the surrounding (lexical) scope at the time they are defined. 
        This simplifies handling "this" in callbacks and nested functions.

              function Counter() {
                this.count = 0;
                setInterval(() => {
                  this.count++; // 'this' correctly refers to the Counter instance
                  console.log(this.count);
                }, 1000);
              }
        
        No "arguments" Object:
        Arrow functions do not have their own arguments object. 
        If you need to access arguments, you can use rest parameters (...args).

        Not Suitable for Constructors or Methods with this:
        Due to their lexical this binding, arrow functions should not be used 
        as constructor functions or as methods where this needs to refer to the object instance.

        Use Cases:
        Arrow functions are commonly used in scenarios where conciseness and 
        lexical this binding are beneficial, such as:
        - Callback functions in array methods (map, filter, reduce).
        - Event handlers.
        - Promises and asynchronous operations.
